Part 1

Tokenisation

Turning text into numbers

Models can't read text — they understand numbers. Tokenisation is the process of breaking text into small pieces (tokens) and mapping each token to a numeric ID. Modern LLMs use Byte-Pair Encoding (BPE), which learns the most efficient subword vocabulary from training data.

Example: "Hello, world! How are you?"

// GPT-4 tokenizer (cl100k_base) "Hello, world! How are you?" ↓ [15496, 11, 1917, 0, 2437, 527, 499, 30] // Decoded back: 15496 → "Hello" 11 → "," 1917 → " world" 0 → "!" 2437 → " How" 527 → " are" 499 → " you" 30 → "?"
100K+
Typical BPE vocabulary size for frontier LLMs (GPT-4: ~100K, Claude: ~150K)
~0.75
Average tokens per English word. "Antidisestablishmentarianism" = ~5 tokens
⚠️
Tokenizer quirks matter: LLMs can't count letters in "strawberry" because they see tokens, not characters

Why Tokenisation Matters

  • Context window is measured in tokens, not words. 128K tokens ≈ 96K English words ≈ a 384-page book
  • Multilingual inequality. "Hello" = 1 token. "こんにちは" = 3 tokens. Non-English languages cost more and fill context faster
  • Code is token-hungry. Python/JSON with lots of whitespace and special chars generates ~2× more tokens than prose
  • Special tokens control model behaviour: <|endoftext|>, <|user|>, <|assistant|>
  • Images become tokens too. Multimodal models like GPT-4o tokenise images into "visual tokens"

💡 Key Insight

Tokenisation is the bridge between human language and machine computation. Every quirk of the tokenizer — from counting errors to multilingual bias — propagates into the model's behaviour.

Part 2

Embeddings

Tokens become meaning

A token ID is just a number — it has no meaning. Embeddings are dense vectors (lists of numbers) that capture the semantic meaning of a token. Words with similar meanings end up close together in "embedding space."

How Embeddings Work

Each token ID looks up a vector from the embedding matrix — a giant lookup table learned during training:

// Token 15496 ("Hello") → Embedding vector (simplified to 8 dims) 15496 → [0.12, -0.45, 0.78, 0.03, -0.91, 0.56, -0.23, 0.67] // Real embedding dimensions: GPT-3: 12,288 dims GPT-4: ~16,384+ dims (estimated) Llama 3:4,096 dims (8B) / 8,192 dims (70B)

Semantic Similarity: Cosine Similarity

Two vectors pointing in the same direction = similar meaning. Cosine similarity measures the angle between vectors:

// These pairs have HIGH cosine similarity: cos("king", "queen") = 0.82 cos("dog", "puppy") = 0.88 cos("happy", "joyful") = 0.91 // These pairs have LOW cosine similarity: cos("king", "banana") = 0.03 cos("run", "elephant")= 0.08

The Famous Analogy: king − man + woman ≈ queen

Embeddings capture semantic relationships as vector arithmetic. The vector from "man" to "king" points in the same direction as the vector from "woman" to "queen" — the royal dimension.

// Vector arithmetic in embedding space: embedding("king") - embedding("man") + embedding("woman") ≈ embedding("queen") ✅ // Other examples: paris - france + italy ≈ rome walking - walk + swim ≈ swimming

What Embeddings Encode

  • Semantic similarity — "cat" and "feline" are close together
  • Analogies — direction vectors capture relationships like gender, tense, capital-of
  • Context. The same word gets different embeddings depending on surrounding text ("bank" as river vs. financial institution)
  • Cross-lingual alignment. "Hello", "Hola", "こんにちは" cluster together despite different tokenizers
  • Multimodal. CLIP-style models embed text and images in the same space — a photo of a dog is close to the word "dog"

💡 Key Insight

Embeddings transform discrete symbols (words) into a continuous, mathematically manipulable space. This is the foundation that makes everything else — attention, generation, reasoning — possible.

Part 3 · The Breakthrough

Attention

"Attention Is All You Need" (2017)

Before attention (RNNs, LSTMs), models processed text sequentially — one word at a time. This was slow and couldn't capture long-range dependencies. Self-attention lets every word look at every other word simultaneously.

The Core Idea: Query, Key, Value

Think of it like a search engine for words:

Query (Q)"What am I looking for?" — the current word's question
Key (K)"What do I contain?" — every word's label
Value (V)"What information do I have?" — the actual content to aggregate

Attention(Q,K,V) = softmax(QKT / √d) × V — the scaled dot-product attention formula

Concrete Example: "The cat sat on the mat because it was tired"

Word "it" needs to know what it refers to. Attention weights show which words "it" is paying attention to:

Multi-Head Attention

Instead of one attention pass, the model runs multiple attention heads in parallel. Each head learns to attend to different types of relationships:

Head 1Syntactic: subject-verb agreement
Head 2Coreference: pronouns → antecedents
Head 3Semantic: related concepts
Head 4Positional: nearby words
Head N... and many more subtle patterns

Why Attention Changed Everything

Before (RNN/LSTM)

  • Sequential — one word at a time
  • Can't parallelize
  • Struggles with long-range context
  • "Vanishing gradient" problem
  • Max effective context: ~50-100 words

After (Attention)

  • Every word ↔ every word simultaneously
  • Fully parallelizable
  • Captures any-distance relationships
  • Stable training even with 1000s of tokens
  • Context: 128K+ tokens

💡 Key Insight

Attention is the fundamental innovation that makes modern LLMs possible. It's how a model knows that "it" refers to "the cat" seven words earlier — and it scales to finding relevant information across a 200-page document.

Part 4 · The Architecture

Transformers

The engine that powers every modern LLM

The Transformer is the neural network architecture introduced in the 2017 paper "Attention Is All You Need." GPT models use a decoder-only variant. The same basic architecture scales from GPT-2 (1.5B params) to GPT-4 (estimated 1.7T params).

GPT Decoder Block (one of N repeated layers)

┌─────────────────────────────────┐ │ INPUT: Token Embeddings │ │ + Positional Encoding │ └───────────┬─────────────────────┘ ↓ ┌─────────────────────────────────┐ │ Masked Multi-Head Attention │ ← "Look at previous words, not future ones" └───────────┬─────────────────────┘ ↓ (Add & LayerNorm) ┌─────────────────────────────────┐ │ Feed-Forward Network (FFN) │ ← Two linear layers with activation └───────────┬─────────────────────┘ ↓ (Add & LayerNorm) ┌─────────────────────────────────┐ │ OUTPUT → Next Decoder Block │ └─────────────────────────────────┘ ↑ Repeated N times ↑
96
GPT-3 decoder layers (175B params). GPT-4: estimated 120 layers, 1.7T params
96
Attention heads per layer in GPT-3 (96 heads × 96 layers = 9,216 total heads)
Masked
Causal masking ensures the model can't "cheat" by looking at future words during training
~60%
FFN layers account for ~⅔ of total parameters (where most "knowledge" is stored)

Key Components Explained

  • Positional Encoding. Attention has no sense of order. Positional encodings inject position information so "dog bites man" ≠ "man bites dog"
  • Masked Self-Attention. During training, each token can only attend to itself and previous tokens — this is what makes it "autoregressive"
  • Feed-Forward Network (FFN). A two-layer fully-connected network applied to each token independently after attention. This is where much of the model's factual knowledge is stored
  • Residual Connections (Add). The input to each sub-layer is added back to its output. This prevents the "vanishing gradient" problem and allows training very deep networks
  • Layer Normalization. Stabilizes training by normalizing activations across the feature dimension

Transformer Variants

ArchitectureUsed ByKey Difference
Decoder-onlyGPT familyAutoregressive — generates left-to-right
Encoder-onlyBERTBidirectional — good for understanding, not generation
Encoder-DecoderT5, BARTUsed for translation, summarization

💡 Key Insight

The Transformer is elegantly simple: attention + feed-forward + residual connections, repeated many times. Its genius is that it's highly parallelizable (all tokens processed simultaneously during training) and scales remarkably well — just add more layers, wider dimensions, and more data.

Training

How models learn from internet-scale text

LLMs learn through a multi-stage process. The core is next-token prediction on trillions of tokens — a surprisingly simple objective that produces remarkably capable models.

Stage 1: Pre-training (The Foundation)

Task: Given "The quick brown", predict "fox"
Data: Internet-scale text (Common Crawl, Wikipedia, books, code, Reddit)
Objective: Minimize prediction error across every position

// The model sees: It predicts: The → predict next token → "quick" (was the model right?) The quick → predict next → "brown" (was the model right?) The quick brown → next → "fox" (was the model right?) // ... across billions of examples, the model learns patterns of language

How RLHF Works

  • 1. Collect preferences. Humans rank multiple model responses to the same prompt (A is better than B)
  • 2. Train a reward model. A separate model learns to predict human preferences — acting as an automated "judge"
  • 3. Fine-tune with RL. The LLM is fine-tuned to maximize the reward model's score, using PPO (Proximal Policy Optimization)
  • 4. The result. Models that are more helpful, refuse harmful requests, and follow instructions — but also more "aligned" to the trainers' values

💡 Key Insight

Pre-training (next-token prediction) gives the model knowledge and language ability. Fine-tuning (SFT + RLHF) makes it useful, safe, and aligned. The trend: more data, more compute, better models — but at exponentially increasing cost.

Part 6 & 7

Inference & Hallucination

How models generate — and why they make things up

Autoregressive Generation (One Token at a Time)

LLMs generate text one token at a time, each step feeding the output back as input for the next prediction. This is why generation feels sequential — it fundamentally is.

Prompt: "The meaning of life is" Step 1: "The meaning of life is" → predict → "to" (prob: 0.42) Step 2: "The meaning of life is to" → predict → "be" (prob: 0.67) Step 3: "The meaning of life is to be" → predict → "happy" (prob: 0.31) // ... continues until the model generates an end-of-sequence token

Controlling Generation: Temperature & Sampling

The model outputs a probability distribution over its entire vocabulary (100K+ tokens). How we sample from it matters:

Temp
Temperature controls randomness. 0 = always pick highest prob (deterministic). 1 = sample proportionally. >1 = more random/creative.
Top-p
Nucleus sampling. Only consider tokens whose cumulative probability reaches p. T=0.7, top-p=0.9 is a common setting.
Top-k
Only sample from the k most likely tokens. Simple but effective. Combined with temperature and top-p in practice.
0→1M
Context window growth: GPT-2 (1K) → GPT-3 (2K) → GPT-4 (128K) → Gemini (1M+). Longer context = more relevant information.

Why LLMs Hallucinate

Hallucination isn't a bug — it's an inevitable consequence of how these models work. They're trained to predict plausible next tokens, not to be factually accurate.

  • Next-token prediction ≠ truth. The model's only objective is to produce statistically probable text. It has no internal "fact checker."
  • Training data is imperfect. The internet contains falsehoods, contradictions, and outdated information. The model learns from all of it.
  • Probabilistic generation. Even with low temperature, the model picks tokens that sound right based on patterns — not because they're verified facts
  • Knowledge cutoffs. Models have a training cutoff date. They don't know what happened after that — but they'll confidently guess

Mitigations

RAG (Retrieval-Augmented Generation)

  • Look up facts in a trusted source before answering
  • Grounds the model in real data
  • Most practical enterprise solution

Other Approaches

  • Chain-of-thought prompting (reason step by step)
  • Constitutional AI (self-critique)
  • Tool use / function calling (delegate to reliable systems)
  • Human-in-the-loop for critical decisions

💡 Key Insight

LLMs are pattern completion engines, not knowledge bases. They excel at tasks where "sounding right" and "being right" overlap (summarization, coding, creative writing). For factual tasks, ground them with RAG, tools, or human verification.

Key Terms & Glossary

Quick reference for LLM fundamentals

Token

A subword unit of text. "Hello world" ≈ 2-3 tokens. Models process text one token at a time.

BPE

Byte-Pair Encoding. The most common tokenisation algorithm. Iteratively merges frequent character pairs.

Embedding

A dense vector (list of numbers) representing the meaning of a token. Similar words are close in embedding space.

Attention

The mechanism that lets each token weigh the importance of every other token. The core innovation behind transformers.

Query / Key / Value

The three matrices in attention. Query="what am I looking for", Key="what do I contain", Value="my actual info".

Multi-Head Attention

Running multiple attention operations in parallel. Each head learns to attend to different types of relationships.

Transformer

The neural network architecture (2017) based entirely on attention. Powers GPT, Claude, Gemini, and Llama.

Decoder-only

Transformer variant used by GPT. Generates text autoregressively (left-to-right). Simpler than encoder-decoder models.

FFN

Feed-Forward Network. The dense layers after attention in each transformer block. Stores much of the model's knowledge.

Positional Encoding

Injects position information into token embeddings. Without it, "dog bites man" = "man bites dog".

Pre-training

The initial training phase on massive text corpora. The model learns language patterns via next-token prediction.

Fine-tuning

Additional training on curated data to make the model helpful, safe, and aligned with human preferences.

RLHF

Reinforcement Learning from Human Feedback. Humans rank outputs → train a reward model → fine-tune the LLM to maximize the reward.

Hallucination

When an LLM generates plausible-sounding but factually incorrect information. An inherent challenge of next-token prediction.

RAG

Retrieval-Augmented Generation. Looking up facts from an external source before generating a response. Reduces hallucination.

Context Window

The maximum number of tokens a model can process at once. GPT-4: 128K. Gemini: 1M+. Determines how much info you can provide.

Temperature

Controls randomness in generation. 0 = deterministic (always same output). 1 = proportional to probabilities. Higher = more creative.

Next-Token Prediction

The core training objective. Given preceding text, predict the most likely next token. Simple task, profound results.

Cosine Similarity

Measures the angle between two embedding vectors. cos(king,queen) = high. cos(king,banana) = low. Range: [-1, 1].

Scaling Laws

Empirical observation that model performance improves predictably with more parameters, data, and compute. "Bigger = better" (to a point).

Softmax

Converts raw scores into a probability distribution (sums to 1). Used to turn attention scores into weights and logits into token probabilities.

Parameter

A learnable weight in the neural network. GPT-3: 175 billion. GPT-4: ~1.7 trillion. More params = more capacity (and cost).

MoE

Mixture of Experts. An architecture where only a subset of parameters activate per token. GPT-4 uses this — 1.7T total, ~220B active per token.

Causal Masking

Prevents the model from seeing future tokens during training. Ensures autoregressive generation — each token only sees the past.
