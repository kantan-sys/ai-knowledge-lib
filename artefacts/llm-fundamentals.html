Part 1

Tokenisation

Turning text into numbers

Models can't read text — they understand numbers. Tokenisation breaks text into small pieces (tokens) and maps each to a numeric ID. Modern LLMs use Byte-Pair Encoding (BPE), which learns the most efficient subword vocabulary.

Example: "Hello, world! How are you?"

// GPT-4 tokenizer (cl100k_base) "Hello, world! How are you?" ↓ [15496, 11, 1917, 0, 2437, 527, 499, 30] 15496 → "Hello" 11 → "," 1917 → " world" 2437 → " How" 527 → " are" 499 → " you"
100K+
Typical BPE vocab size (GPT-4: ~100K, Claude: ~150K)
~0.75
Avg tokens per English word
⚠️
LLMs see tokens not letters — can't count letters in "strawberry"
$
API pricing per token — tokenisation affects costs

Why Tokenisation Matters

  • Context window is in tokens. 128K tokens ≈ 96K English words ≈ a 384-page book
  • Multilingual inequality. "Hello" = 1 token. "こんにちは" = 3 tokens. Non-English costs more
  • Code is token-hungry. Python/JSON whitespace & special chars → ~2× more tokens than prose
  • Special tokens control behaviour: <|endoftext|>, <|user|>, <|assistant|>
  • Images become tokens too. GPT-4o tokenises images into "visual tokens"

💡 Key Insight

Tokenisation is the bridge between human language and machine computation. Every quirk — from counting errors to multilingual bias — propagates into the model's behaviour.

Embeddings — From Numbers to Meaning

Tokens become vectors in semantic space

A token ID like 15496 is just a number — it has no meaning on its own. Embeddings give each token meaning by converting it into a vector — a list of hundreds or thousands of numbers. Think of it like giving every word coordinates on a map. Words with similar meanings end up close together.

Part 3 · The Breakthrough

Attention

"Attention Is All You Need" (2017)

Before attention (RNNs, LSTMs), models processed text sequentially — one word at a time. This was slow and couldn't capture long-range dependencies. Self-attention lets every word look at every other word simultaneously.

The Core Idea: Query, Key, Value

Think of it like a search engine for words:

Query (Q)"What am I looking for?" — the current word's question
Key (K)"What do I contain?" — every word's label
Value (V)"What information do I have?" — the actual content to aggregate

Concrete Example: "The cat sat on the mat because it was tired"

Word "it" needs to know what it refers to:

Multi-Head Attention

Multiple attention heads run in parallel — each learns different types of relationships:

Head 1Syntactic: subject-verb agreement
Head 2Coreference: pronouns → antecedents
Head 3Semantic: related concepts
Head 4Positional: nearby words
Head N…many more subtle patterns

Why Attention Changed Everything

Before (RNN/LSTM)

  • Sequential — one word at a time
  • Can't parallelize
  • Max effective context: ~50-100 words
  • "Vanishing gradient" problem

After (Attention)

  • Every word ↔ every word simultaneously
  • Fully parallelizable
  • Context: 128K+ tokens
  • Stable training at any distance

💡 Key Insight

Attention is how a model knows that "it" refers to "the cat" seven words earlier — and it scales to finding relevant information across a 200-page document.

Transformers

The engine powering every modern LLM

The Transformer architecture (2017 paper) powers GPT, Claude, Gemini, and Llama. GPT uses a decoder-only variant. The same architecture scales from GPT-2 (1.5B params) to GPT-4 (~1.7T params).

GPT Decoder Block (one of N repeated layers)

┌──────────────────────────┐ │ INPUT: Token Embeddings │ │ + Positional Encoding │ └────────┬─────────────────┘ ↓ ┌──────────────────────────┐ │ Masked Multi-Head Attn │ ← "Look at previous words only" └────────┬─────────────────┘ ↓ (Add & LayerNorm) ┌──────────────────────────┐ │ Feed-Forward Network │ ← Two linear layers, ~60% of params └────────┬─────────────────┘ ↓ (Add & LayerNorm) ┌──────────────────────────┐ │ OUTPUT → Next Block │ └──────────────────────────┘
96
GPT-3 layers × 96 heads/layer. GPT-4: ~120 layers, 1.7T params
~60%
FFN layers account for ~⅔ of total parameters — where knowledge is stored

Key Components

  • Positional Encoding. Attention has no sense of order. Positional encodings inject position info so "dog bites man" ≠ "man bites dog"
  • Masked Self-Attention. Each token only attends to itself and previous tokens — makes generation autoregressive
  • Feed-Forward Network. Two-layer network after attention. Where most factual knowledge is stored
  • Residual Connections. Input added back to output of each sub-layer. Prevents vanishing gradients in deep networks
  • Layer Normalization. Stabilizes training by normalizing activations

Transformer Variants

ArchitectureUsed ByKey Difference
Decoder-onlyGPT familyAutoregressive — generates left-to-right
Encoder-onlyBERTBidirectional — good for understanding
Encoder-DecoderT5, BARTTranslation, summarization

💡 Key Insight

The Transformer is elegantly simple: attention + feed-forward + residual connections, repeated many times. Its genius: fully parallelizable and scales remarkably — just add more layers, wider dimensions, and more data.

Training

How models learn from internet-scale text

LLMs learn through multi-stage training. The core is next-token prediction on trillions of tokens — a surprisingly simple objective that produces remarkably capable models.

Stage 1: Pre-training

Task: Given "The quick brown", predict "fox"
Data: Internet-scale text (Common Crawl, Wikipedia, books, code)
Objective: Minimize prediction error across every position

// The model sees: It predicts: The → predict next token → "quick" The quick → predict next → "brown" The quick brown → next → "fox" // ...across billions of examples, patterns of language emerge

How RLHF Works

  • 1. Collect preferences. Humans rank multiple model responses (A is better than B)
  • 2. Train a reward model. Learns to predict human preferences — an automated "judge"
  • 3. Fine-tune with RL. LLM optimised via PPO to maximise the reward model's score
  • 4. Result. Models that are helpful, safe, refuse harmful requests — aligned to trainer values

Scaling Laws

GPT-2 (2019)1.5B40GB text

💡 Key Insight

Pre-training gives the model knowledge and language. Fine-tuning (SFT + RLHF) makes it useful, safe, and aligned. The trend: more data, more compute, better models — at exponentially increasing cost.

Part 6 & 7

Inference & Hallucination

How models generate — and why they make things up

Autoregressive Generation (One Token at a Time)

LLMs generate one token at a time, each step feeding output back as input for the next prediction.

Prompt: "The meaning of life is" Step 1: "The meaning of life is" → predict → "to" (42%) Step 2: "The meaning of life is to" → predict → "be" (67%) Step 3: "...is to be" → predict → "happy" (31%) // Continues until end-of-sequence token generated

Controlling Generation

Temp
Temperature: 0 = deterministic, 1 = proportional, >1 = more creative/random
Top-p
Nucleus sampling: only consider tokens whose cumulative probability reaches p
Top-k
Only sample from the k most likely tokens. Combined with temperature and top-p
0→1M
Context window: GPT-2 (1K) → GPT-4 (128K) → Gemini (1M+). Longer = more info

Why LLMs Hallucinate

Hallucination isn't a bug — models are trained to predict plausible text, not to verify facts.

  • Next-token prediction ≠ truth. No internal fact-checker. Only objective: produce statistically probable text
  • Training data is imperfect. Internet contains falsehoods, contradictions, outdated info
  • Probabilistic generation. Picks tokens that sound right based on patterns — not verified facts
  • Knowledge cutoffs. Models don't know what happened after training — but will confidently guess

Mitigations

RAG

  • Look up facts in trusted source first
  • Grounds model in real data
  • Most practical enterprise solution

Other Approaches

  • Chain-of-thought prompting
  • Constitutional AI (self-critique)
  • Tool use / function calling
  • Human-in-the-loop

💡 Key Insight

LLMs are pattern completion engines, not knowledge bases. For factual tasks, ground them with RAG, tools, or human verification. They excel where "sounding right" and "being right" overlap.

Reference

Key Terms & Glossary

Quick reference for LLM fundamentals

Token

A subword unit of text. "Hello world" ≈ 2-3 tokens. Models process text one token at a time.

BPE

Byte-Pair Encoding. The most common tokenisation algorithm. Iteratively merges frequent character pairs.

Embedding

A dense vector (list of numbers) representing the meaning of a token. Similar words are close in embedding space.

Attention

The mechanism that lets each token weigh the importance of every other token. Core innovation behind transformers.

Query / Key / Value

The three matrices in attention. Query="what am I looking for", Key="what do I contain", Value="my actual info".

Multi-Head Attention

Running multiple attention operations in parallel. Each head learns different types of relationships.

Transformer

The neural network architecture (2017) based entirely on attention. Powers GPT, Claude, Gemini, and Llama.

Decoder-only

Transformer variant used by GPT. Generates text autoregressively (left-to-right).

FFN

Feed-Forward Network. Dense layers after attention in each block. Stores much of the model's knowledge.

Positional Encoding

Injects position information into embeddings. Without it, "dog bites man" = "man bites dog".

Pre-training

Initial training on massive text corpora. Model learns language patterns via next-token prediction.

Fine-tuning

Additional training on curated data to make the model helpful, safe, and aligned.

RLHF

Reinforcement Learning from Human Feedback. Humans rank outputs → train reward model → fine-tune LLM.

Hallucination

When an LLM generates plausible-sounding but factually incorrect information. Inherent to next-token prediction.

RAG

Retrieval-Augmented Generation. Looking up facts from an external source before generating. Reduces hallucination.

Context Window

Maximum tokens a model can process at once. GPT-4: 128K. Gemini: 1M+. Determines how much info you can provide.

Temperature

Controls randomness in generation. 0 = deterministic. 1 = proportional. Higher = more creative.

Next-Token Prediction

The core training objective. Given preceding text, predict the most likely next token. Simple task, profound results.

Cosine Similarity

Measures the angle between two embedding vectors. cos(king,queen) = high. cos(king,banana) = low. Range: [-1, 1].

Semantic Space

The high-dimensional space where embeddings live. Similar concepts cluster together. Enables vector arithmetic.

Embedding Matrix

A giant lookup table that maps each token ID to its vector. Learned during training. Size: vocab × dimensions.

Softmax

Converts raw scores into a probability distribution. Used for attention weights and token probability selection.

Parameter

A learnable weight. GPT-3: 175 billion. GPT-4: ~1.7 trillion. More params = more capacity (and cost).

MoE

Mixture of Experts. Only a subset of parameters activate per token. GPT-4: 1.7T total, ~220B active per token.

Causal Masking

Prevents seeing future tokens during training. Ensures autoregressive generation — each token only sees the past.
